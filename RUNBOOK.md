## Current onsite entry points

The checked-in one-click script is currently bound to the 2026-07-24 v12 map
and `shared-board-odin1-20260723-v3` session. It is safe to use only while
those identifiers still match the physical placement.

No-motion full-stack check:

```bash
bash hub/scripts/realworld_oneclick.sh \
  --mode debug \
  --scene-id debug-chair \
  --goal-category chair
```

This starts/validates the Hub, GLM, current maps, Foxglove and read-only
receivers, then runs one real VLM shadow. It must end with
`DEBUG_FULLSTACK_READY`, Hub `GOAL=false`, no WSJ bridge and no live receiver.

Physical episode, only after the debug result and a fresh onsite confirmation:

```bash
bash hub/scripts/realworld_oneclick.sh \
  --mode live \
  --scene-id scene01-chair \
  --episode-id scene01-chair-run01 \
  --goal-category chair \
  --operator-confirmation OPERATOR_PRESENT_AND_ROBOTS_CLEAR
```

The confirmation is single-use. Cleanup, failure, restart or power loss
consumes it. The script must restore debug receivers and `GOAL=false` on every
exit path.

Current limitation: calibration capture and map/session identifiers are not
yet generated by this script; they are prerequisites. Do not edit hard-coded
IDs silently. The operator-workflow refactor must persist one session manifest
and make debug/live consume that exact manifest.

## Local environment finding

The host has Python 3.10 and PyTorch `2.7.1+cu126` with `torch.cuda.is_available() == True` on the A6000.  `transformers`, `accelerate`, OpenAI `clip`, OpenCV, and Uvicorn are present; `bitsandbytes`, FastAPI, OpenAI client, Ultralytics, Detectron2, Habitat, Habitat-Sim, scikit-image, and yacs are not.  This supports a clean local hub environment; it is not a reason to import the HPC overlay.

Create a dedicated local environment or Docker image only after the model transfer completes.  Start with the VLM service dependencies and a robot-ingest replay, and leave Detectron2/Habitat out of the first real-robot path unless a retained module demonstrably needs them.
